# 荒疫模式 (Blight Mode) - 总体设计概览

## 1. Mod简介
本Mod名为“荒疫模式（Blight Mode）”，旨在为《Slay the Spire 2》添加一个大型独立游戏模式。
它具有独立的难度体系、卡牌附魔堆叠机制、新的奖励类型以及专属的遗物和特殊卡牌。

**核心特色：**
1. **全新模式与难度：** 在默认、每日、自定义之外，新增“荒疫模式”，且支持多人联机。包含独立的 0阶段 至 5阶段 难度系统（对应原版进阶10以上的强化幅度）。
2. **多重附魔与叠加（核心机制）：** 突破原版的单附魔限制！卡牌可以重复附魔，甚至叠加多种不同的附魔。
3. **全新的奖励池系统：** 卡牌战利品直接带有随机附魔（按稀有度池划分）；全新的“锻造奖励”（三选一）。
4. **强化的篝火休息：** 恢复更多生命值，增强玩家在更高危险境地的生存能力。
5. **怪物意图全面重构：** 所有敌人的AI意图和强化在荒疫模式下将发生显著改变。

## 2. 技术可行性与实现策略
经过对 `MegaCrit.Sts2.Core` 的反编译源码分析，本Mod的核心功能可行性如下：

- **新增游戏模式 (Game Mode) 和 进阶难度 (Ascension)**：
  因为原版的 `GameMode` 和 `AscensionLevel` 被实现为硬编码的 `enum`（枚举），直接继承可能较难。
  *策略：* 需要使用 Harmony 进行 IL Patch。我们可以拦截 UI 生成代码，在模式选择列表中强行植入“荒疫模式”按钮。当进入“荒疫模式”时，我们可以在全局使用静态标记（如 `BlightMod.IsBlightMode = true`），并在原版检查难度的底层用 Patch 进行替换。
  联机支持通常也会绑定在 `GameMode`，需要确认原版联机的房间校验机制是否允许自定义参数。

- **多重附魔与叠加系统（Enchantment）**：
  原版 `EnchantmentModel` 机制较为完善。
  *策略：* 我们需要拦截卡牌在试图应用新附魔时的判断逻辑（覆盖 `CardModel` 中关于附魔是否已存在的检查），允许在列表中推入多个相同的 `EnchantmentModel` 或使其动态变量叠加。（详见 \`02_Enchantment_System.md\`）

- **新增“锻造奖励”和随机附魔掉落**：
  原版的奖励基于 `Reward`（如 `GoldReward`, `CardReward` 等）。
  *策略：* 我们可以继承 `Reward` 创建 `ForgeReward`，然后重写奖励生成逻辑，在战胜后向战利品列表加入该对象。（详见 \`03_Rewards_and_Campfire.md\`）

- **篝火改动与怪物 AI**：
  篝火的选项类为 `RestSiteOption`（如 `HealRestSiteOption`）。
  *策略：* 使用 Harmony Prefix 拦截篝火生命恢复计算逻辑。对于怪物AI，使用 Patch 拦截 `MonsterModel.TakeTurn()` 或是意图生成，使其在 `BlightMod.IsBlightMode == true` 时使用一套我们全新定义的意图逻辑。
